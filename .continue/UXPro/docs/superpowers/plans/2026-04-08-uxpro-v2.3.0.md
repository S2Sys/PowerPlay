                    SECTION 27: ANIMATION LIBRARY PATTERNS
═══════════════════════════════════════════════════════════════════════════════

### 27.1 Animation Library Fundamentals

**Purpose:** Reusable animation systems that can be composed, sequenced, and controlled programmatically.

**Core Principle:** Animations should be:
1. Declarable (defined once, reused many times)
2. Composable (combined with other animations)
3. Queryable (can check current state)
4. Cancellable (can interrupt mid-animation)

**Implementation Pattern:**

```css
/* Define animation library as reusable keyframes */
@keyframes slide-in-left {
  from {
    opacity: 0;
    transform: translateX(-100%);
  }
  to {
    opacity: 1;
    transform: translateX(0);
  }
}

@keyframes fade-in {
  from { opacity: 0; }
  to { opacity: 1; }
}

@keyframes scale-pop {
  0% { transform: scale(0.8); opacity: 0; }
  50% { transform: scale(1.05); }
  100% { transform: scale(1); opacity: 1; }
}

/* Apply with custom timing */
.animate-slide-in {
  animation: slide-in-left 0.4s cubic-bezier(0.34, 1.56, 0.64, 1) forwards;
}

.animate-fade {
  animation: fade-in 0.3s ease-out forwards;
}

.animate-pop {
  animation: scale-pop 0.5s cubic-bezier(0.34, 1.56, 0.64, 1) forwards;
}
```

**Best Practices:**
- Use `forwards` fill mode to maintain end state
- Combine opacity changes with transforms (GPU-accelerated)
- Always provide prefers-reduced-motion alternative
- Keep animation duration under 500ms for UI feedback
```

### 29.1 State Machine UI

**Purpose:** Visually represent discrete application states and their valid transitions.

**Order Processing State Machine Example:**

  </div>
  
  <div class="state-connector state-active"></div>
  
  <div class="state-step" data-state="confirmed" data-active="true">

    <div class="state-timestamp">1 min ago</div>
  </div>
  
  <div class="state-connector state-pending"></div>
  
  <div class="state-step" data-state="shipped" data-active="false">
    <div class="state-badge">Shipped</div>
    <div class="state-timestamp">—</div>
  </div>
  
  <div class="state-connector state-disabled"></div>
  
  <div class="state-step" data-state="delivered" data-active="false">
    <div class="state-badge">Delivered</div>
    <div class="state-timestamp">—</div>
  </div>
</div>

<!-- Allowed transitions -->
<div class="state-actions">
  <button class="btn btn-secondary" disabled title="Cannot cancel once shipped">Cancel Order</button>
  <button class="btn btn-primary">View Tracking</button>
</div>
