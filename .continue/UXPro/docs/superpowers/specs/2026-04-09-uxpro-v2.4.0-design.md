## Section 30: Form Builder Patterns

### Purpose
Teach AI tools to generate dynamic, adaptive forms that respond to user input and handle complex workflows. Beyond static input collections, this section covers conditional visibility, multi-step wizards, field dependencies, custom validation, and state management patterns.

### Coverage Scope

#### Basic Dynamic Forms
- **Conditional field visibility:** Show/hide fields based on user selections (e.g., "Ship to different address?" checkbox reveals address fields)
- **Simple dependencies:** One field value affects another field's options (e.g., country selection populates state dropdown)
- **Section grouping:** Organize related fields into expandable sections
- **Basic validation feedback:** Show errors/success states inline

#### Advanced Form Systems
- **Multi-step wizards:** Break forms into numbered steps with progress tracking, validation per step, next/previous navigation
- **Complex field dependencies:** Role selection determines which permission checkboxes appear; date range selection validates end date >= start date
- **Custom validation rules:** Beyond HTML5 validation (e.g., "password must contain uppercase, number, special char")
- **Field groups and arrays:** Add/remove field groups (e.g., add multiple addresses, phone numbers)
- **Save/restore state:** Save form progress, allow returning to incomplete forms
- **Error summarization:** Collect all validation errors and display at top with anchor links to fields

#### Accessibility Requirements
- Conditional fields use `aria-hidden` and semantic markup
- Validation messages announced via ARIA live regions
- Keyboard navigation through all form elements and buttons
- Focus management when showing/hiding fields
- Error messages linked to form fields via `aria-describedby`
- Step indicators properly labeled for screen readers

#### Mobile Considerations
- Single-column layout on mobile
- Large touch targets (48×48px minimum)
- Mobile-optimized wizard steps (stacked vertically)
- Keyboard visibility on mobile (avoid covering inputs)

### Design Tokens for Forms
```
Form Step/Wizard Tokens:
- --form-step-indicator-bg: Background for visited/current/pending steps
- --form-step-current-color: Color for current step indicator
- --form-progress-bar-color: Progress bar fill color
- --form-progress-bar-bg: Progress bar background

Validation State Tokens:
- --form-error-bg: Error field background (light red)
- --form-error-border: Error field border color (red, 4.5:1 contrast minimum)
- --form-error-text: Error message text color (red, 4.5:1 contrast minimum)
- --form-warning-bg: Warning field background (light amber)
- --form-warning-border: Warning field border color
- --form-success-bg: Success field background (light green)
- --form-success-border: Success field border color

Conditional Visibility Tokens:
- --form-transition-duration: Fade-in/fade-out animation duration (200-300ms)
- --form-transition-easing: Animation easing curve (ease-in-out)
- --form-conditional-opacity: Opacity for hidden-but-present fields

Dynamic Field Tokens:
- --form-field-add-btn-bg: Add field/group button background
- --form-field-remove-btn-bg: Remove field/group button background (light red)
- --form-group-border: Border for field groups
- --form-group-spacing: Spacing between field groups
```

## Section 31: Advanced Data Table Interactions

### Purpose
Teach AI tools to generate data tables with production-grade interactivity. Beyond displaying static data, this section covers sorting, filtering, pagination, real-time updates, row selection, inline editing, and responsive behavior for enterprise applications.

### Coverage Scope

#### Static Data Patterns
- **Column sorting:** Click column header to sort ascending/descending, multi-column sort with shift+click, visual indicator showing sort direction
- **Filtering:** Text search across all/specific columns, select dropdowns for categorical data, date range pickers, combined filters, clear all filters button
- **Pagination:** Page number buttons (1, 2, 3...), "Previous/Next" buttons, "Rows per page" selector, total row count display, jump to page input
- **Column visibility:** Toggle checkboxes to show/hide columns, save user preference (localStorage)
- **Sorting accessibility:** Announce sort direction to screen readers, keyboard accessible sort controls
- **Touch-friendly pagination:** Large tap targets for mobile users

#### Real-Time Data Patterns
- **Adding new rows:** New data appears at top/bottom without page reload, "New X rows added" notification
- **Updating existing rows:** Row updates in place, optional "updated 2 seconds ago" indicator, animation to highlight changed row
- **Removing rows:** Row disappears smoothly (fade-out animation), confirmation optional for bulk delete
- **Loading states:** Loading spinner or skeleton during fetch, disable sort/filter during update
- **Conflict resolution:** Handle case where data is deleted server-side while user is viewing it
- **Network error handling:** Show error message, retry button, fallback behavior

#### Advanced Patterns
- **Row selection:** Checkboxes for individual row selection, "select all" checkbox in header, selected row count display
- **Bulk actions:** Action bar appears when rows selected (delete, export, archive), applies only to selected rows
- **Inline editing:** Click cell to edit (text, select, date input), save inline (no page reload), undo last edit, validation on save
- **Responsive table behavior:** On mobile, switch from table to card layout showing key fields, horizontal scroll with sticky first column, collapsible rows
- **Virtual scrolling:** For 1000+ rows, render only visible rows (performance optimization)

#### Accessibility Requirements
- Keyboard navigation through table (arrow keys to move, Tab to action buttons)
- Focus visible on all interactive elements
- Announcements for sort changes, new rows, deleted rows via ARIA live regions
- Table header properly marked with `<th>` and `scope` attribute
- Row selection announced (e.g., "3 of 50 rows selected")
- Inline edit fields properly labeled and focused
- Mobile: Touch targets 48×48px minimum

#### Performance Considerations
- Pagination for large datasets (don't render 10,000 rows at once)
- Virtual scrolling for datasets exceeding 1000 rows
- Debounce search input (wait 300ms after typing stops before filtering)
- Optimize re-renders (use `key` attributes in examples, discuss performance in comments)

## Section 32: Micro-app Shell Patterns

### Purpose
Teach AI tools to generate consistent, accessible multi-page app structures (shells/layouts). This section covers navigation patterns ranging from simple flat menus to complex nested hierarchies, responsive behavior from mobile to desktop, and state management for multi-page applications.

### Coverage Scope

#### Simple Shells
- **Header/Top Navigation:** Logo/branding area, main navigation links, user menu (dropdown with Profile/Settings/Logout)
- **Sidebar Navigation:** Flat menu with 5-8 items, icons + labels, active indicator for current page, fixed width (240-280px)
- **Main Content Area:** Flexible, grows to fill available space, padding for comfortable reading (32-48px)
- **Footer:** Copyright, links section, stays at bottom (sticky or absolute)
- **Responsive behavior:** On mobile (<768px), sidebar collapses to hamburger menu (drawer), top nav remains visible

#### Complex Shells
- **Nested Navigation Menus:** Multi-level hierarchies (e.g., Settings → Account → Profile), expand/collapse with chevron indicators
- **Breadcrumbs:** Show page hierarchy (Home > Settings > Account > Profile), clickable to navigate back
- **Active state indicators:** Current page highlighted in navigation, ancestors also highlighted to show position in hierarchy
- **Search functionality:** Search box in header for finding pages/content
- **User profile section:** In sidebar footer, showing user avatar, name, status (Online/Offline)

#### Responsive Behavior
- **Desktop (1024px+):** Full sidebar visible, content area takes remaining space
- **Tablet (768px-1023px):** Sidebar visible but narrower, collapsible to icon-only mode
- **Mobile (<768px):** Hamburger menu opens drawer sidebar, overlay darkens main content, drawer slides from left
- **Drawer animation:** Smooth slide-in (300-400ms), closes on item selection or outside click
- **Sticky header:** Header remains visible on scroll, sidebar content scrolls independently
- **Touch targets:** All navigation items 48×48px minimum (including icon-only state)

#### Shell Behaviors
- **Active menu highlighting:** Current page link highlighted with background color or underline
- **Navigation state persistence:** Remember which sections user has expanded (localStorage)
- **Keyboard navigation:** Tab through menu items, arrow keys to move between items, Enter/Space to activate submenu toggle
- **Focus management:** Focus moves to first menu item when drawer opens, returns to hamburger when drawer closes
- **Animation:** Menu expand/collapse animates smoothly, drawer slides in from side
- **Dark mode:** Full dark mode support for all shell elements

#### Accessibility Requirements
- Semantic HTML: `<nav>`, `<ul>`, `<li>` for menus, `<header>`, `<main>`, `<footer>` for layout sections
- ARIA labels: `aria-label` for icon-only navigation, `aria-current="page"` for active menu item
- Keyboard navigation: All interactive elements reachable via Tab, arrow keys work in menus
- Focus visible: All focused elements have visible focus indicator (3:1 contrast minimum)
- Screen reader announcements: Navigation announced (e.g., "Navigation menu, 8 items"), active page announced
- Drawer behavior: When drawer opens, focus moves inside, tab stays within drawer
- Semantic link structure: Links are actual `<a>` tags (not divs with click handlers)

#### Integration Patterns
- **Page content adaptation:** Shell stays fixed, only main content area changes between pages
- **Nested page layout:** Breadcrumbs update, menu items highlight appropriately
- **Loading states:** Show loading indicator in main area while page loads, skeleton content optional
- **Error states:** Error message in main area if page load fails, retry button available

### Design Tokens for Shell

- --shell-sidebar-width: Sidebar width on desktop (240-280px)
- --shell-sidebar-width-narrow: Narrow sidebar width on tablet (60-80px, icon-only mode)

- --shell-nav-active-bg: Active menu item background
- --shell-nav-active-text: Active menu item text color

- --shell-nav-icon-size: Navigation icon size (20-24px)
