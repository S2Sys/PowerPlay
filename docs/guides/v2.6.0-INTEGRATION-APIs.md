## 🎯 Event-Driven Architecture

**Core Principle**: Decouple services via events. Services react to what happened, not instructions.

### Event Sourcing

**Idea**: Store all state changes as immutable events.

```
Traditional: Update record in database
User → UpdateOrder(id=123, status="Paid")
Database: Orders { id: 123, status: "Paid" }  ← Single record

Event Sourcing: Append event to log
User → OrderCreated(id=123, amount=$100)
       PaymentProcessed(id=123, txn="abc")
       OrderShipped(id=123, date=today)
Event Log:
  - OrderCreated(...) ← Immutable
  - PaymentProcessed(...) ← Immutable
  - OrderShipped(...) ← Immutable
  
Reconstruct state: Replay all events → Order is "Shipped"
```

**Benefits**:
- Complete audit trail (every change recorded)
- Can replay to recover state
- Can go back in time (what was order status on date X?)
- Naturally supports CQRS

**Implementation**:
```csharp
// Event sourcing - store immutable events
public class Order
{
    public Guid Id { get; set; }
    public List<IEvent> Events { get; set; } = new();  // Immutable log
    
    // Reconstruct state from events
    public OrderStatus GetStatus()
    {
        return Events
            .OfType<StateChangeEvent>()
            .LastOrDefault()
            ?.Status ?? OrderStatus.Created;
    }
}

// Add event (NEVER update event log)
public void ProcessPayment(decimal amount)
{
    Events.Add(new PaymentProcessedEvent
    {
        OrderId = Id,
        Amount = amount,
        Timestamp = DateTime.UtcNow
    });
    // Event appended (immutable) — never update/delete
}
```

### CQRS (Command Query Responsibility Segregation)

**Idea**: Separate write model from read model.

```
Traditional:
Request → Database (normalize)
         ↓ (join all tables)
Response ← Slow, complex queries

CQRS:
Commands (Write):
  CreateOrder → Normalized database (3NF)
               ↓ (emit event)
Events:
  OrderCreated ↓
  ↓
Queries (Read):
  GetOrder → Denormalized view (optimized)
            ↓ (flat, simple queries)
  Response ← Fast, simple queries
```

**Example**:
```csharp
// COMMAND SIDE (write model) — normalized
public async Task CreateOrderAsync(CreateOrderCommand cmd)
{
    // Validate
    if (cmd.Amount <= 0) throw new Exception("Invalid amount");
    
    // Emit event (only source of truth)
    var @event = new OrderCreatedEvent
    {
        OrderId = Guid.NewGuid(),
        CustomerId = cmd.CustomerId,
        Amount = cmd.Amount
    };
    
    await _eventStore.AppendAsync(@event);  // Append-only
}

// QUERY SIDE (read model) — denormalized for speed
public async Task<OrderDto> GetOrderAsync(Guid orderId)
{
    // Simple query (no joins)
    return await _db.Orders
        .AsNoTracking()
        .Where(o => o.OrderId == orderId)
        .SingleOrDefaultAsync();
}

// Event handler updates read model asynchronously
public class OrderCreatedEventHandler : IConsumer<OrderCreatedEvent>
{
    public async Task Handle(OrderCreatedEvent @event)
    {
        // Insert into read model (eventually consistent)
        await _readDb.Orders.AddAsync(new OrderReadModel
        {
            OrderId = @event.OrderId,
            CustomerId = @event.CustomerId,
            Amount = @event.Amount,
            Status = "Created"
        });
        await _readDb.SaveChangesAsync();
    }
}
```

### Saga Pattern (Distributed Transactions)

**Idea**: Multi-step workflows with compensating transactions (rollback on failure).

```
Order → Payment → Shipment
  ↓        ↓         ↓
Create   Charge    Ship

If payment fails:
  ↓
Cancel order (compensating action)
Refund (if already charged)
```

**Example**:
```csharp
// Saga orchestrator
public class OrderSaga
{
    public async Task ProcessOrderAsync(Order order)
    {
        try
        {
            // Step 1: Create order
            await _orderService.CreateAsync(order);
            
            // Step 2: Process payment
            await _paymentService.ChargeAsync(order.Id, order.Amount);
            
            // Step 3: Ship order
            await _shipmentService.ShipAsync(order.Id);
        }
        catch (PaymentException ex)
        {
            // Compensating action: cancel order
            await _orderService.CancelAsync(order.Id);
            throw;
        }
    }
}
```

## 🛣️ API Gateway Patterns

**Purpose**: Single entry point for all client requests.

### Request Routing

```
Client request:
GET /api/orders/123

API Gateway:
  ↓ Route by path
  ├─ /api/orders/* → OrderService
  ├─ /api/payments/* → PaymentService
  └─ /api/users/* → UserService

OrderService:
  GET /orders/123
  ↓
Response back through gateway
```

**Implementation** (using Ocelot):
```yaml
Routes:
  - UpstreamPathTemplate: /api/orders/{everything}
    DownstreamPathTemplate: /orders/{everything}
    DownstreamHostAndPorts:
      - Host: order-service
        Port: 80
    RateLimitOptions:
      EnableRateLimiting: true
      Period: "1m"
      Limit: 100
```

### API Composition

**Idea**: Aggregate responses from multiple services in one call.

```
Client request:
GET /api/orders/123/complete

API Gateway:
  Call Order Service (parallel)   ─┐
  Call Payment Service (parallel) ─┼→ Aggregate → Response
  Call User Service (parallel)    ─┘
```

**Example**:
```csharp
[HttpGet("orders/{id}/complete")]
public async Task<OrderCompleteDto> GetCompleteOrderAsync(Guid id)
{
    // Call services in parallel
    var order = await _orderService.GetAsync(id);
    var payments = await _paymentService.GetByOrderAsync(id);
    var user = await _userService.GetAsync(order.CustomerId);
    
    // Aggregate response
    return new OrderCompleteDto
    {
        Order = order,
        Payments = payments,
        Customer = user
    };
}
```

### Rate Limiting

**Idea**: Prevent abuse, fair usage.

```
User A: 50 requests/minute (allowed)
User B: 150 requests/minute (over 100 limit) → 429 Too Many Requests

Each endpoint has different limits:
GET /orders     → 100 req/min
POST /payments  → 50 req/min (stricter)
```

**Implementation**:
```csharp
[HttpGet("orders")]
[RateLimit(100, "1m", PerUser = true)]  // 100 per minute per user
public async Task<List<Order>> GetOrdersAsync()
{
    return await _db.Orders.ToListAsync();
}
```

### Caching

**Idea**: Reduce backend load, improve latency.

```
GET /api/orders/123
  ├─ Cache hit? → Return cached response (5ms)
  └─ Cache miss?
      ├─ Call OrderService
      ├─ Cache response for 5 min
      └─ Return response (100ms)

PUT /api/orders/123  (update order)
  ├─ Update backend
  ├─ Invalidate cache (delete cache entry)
  └─ Return response
```

## 📨 Message Queues

**Purpose**: Async communication between services.

### Queues vs. Topics

**Queue** (Point-to-Point):
```
Producer → Queue → Consumer
  Order Processing Task
  (only one service processes)
```

**Topic** (Publish-Subscribe):
```
Producer → Topic → Subscriber 1
             ↓  → Subscriber 2
             → Subscriber 3
  Order Created Event
  (multiple services react)
```

### RabbitMQ Patterns

**Setup**:
```yaml
# Queues (point-to-point)
queues:
  order_processing:
    durable: true  # Survive broker restart
    deadLetterExchange: dlx  # Failed messages → DLQ

# Topics (publish-subscribe)
exchanges:
  order_events:
    type: topic
    bindings:
      - routing_key: "order.#"  # Match order.*
        queue: email_notifications
```

**Producer**:
```csharp
await _messageQueue.PublishAsync("order_events", 
    new OrderCreatedEvent { OrderId = 123, Amount = 100 });
```

**Consumer**:
```csharp
public class SendEmailConsumer : IConsumer<OrderCreatedEvent>
{
    public async Task Consume(ConsumeContext<OrderCreatedEvent> context)
    {
        await _emailService.SendAsync(context.Message.OrderId);
        // Message auto-acknowledged (removed from queue)
    }
}
```

### Dead Letter Queue (DLQ)

**Idea**: Route failed messages to separate queue for analysis.

```
Message → Consumer
  ├─ Success → Acknowledge (remove from queue)
  └─ Failure (after 5 retries)
      └─ → Dead Letter Queue
          (alert ops, investigate)
```

**Retry Logic**:
```csharp
// Exponential backoff: 1s, 2s, 4s, 8s, 16s (max 5 retries)
var retryDelays = new[] { 1, 2, 4, 8, 16 }.Select(s => TimeSpan.FromSeconds(s));

try
{
    await ProcessMessageAsync(message);
}
catch (Exception ex)
{
    if (retryCount < 5)
    {
        var delay = retryDelays[retryCount];
        await _queue.DeferAsync(message, delay);  // Retry later
    }
    else
    {
        await _dlq.SendAsync(message);  // Give up, send to DLQ
    }
}
```

## 🪝 Webhooks

**Purpose**: Push real-time events to external systems.

### Reliable Delivery

```
Event occurs
  ↓
Create webhook payload
  ↓
Sign with HMAC-SHA256
  ↓
POST to subscriber URL
  ├─ 2xx? → Success
  └─ Error? → Retry with backoff
      ├─ Retry 1: 1 second (wait)
      ├─ Retry 2: 2 seconds
      ├─ Retry 3: 4 seconds
      ├─ Retry 4: 8 seconds
      ├─ Retry 5: 16 seconds
      └─ Max retries: 5 → Disable webhook, alert
```

### Signature Verification

**Subscriber side** (verify webhook is genuine):

```csharp
[HttpPost("webhooks/smartworkz")]
public async Task<IActionResult> ReceiveWebhookAsync()
{
    var body = await GetBodyAsync();  // Read request body
    var signature = Request.Headers["X-Signature"];
    
    // Verify signature
    var expected = ComputeHMAC(body, _secret);
    if (signature != expected)
        return Unauthorized();  // Spoofed webhook
    
    // Check timestamp (not > 5 min old)
    var timestamp = Request.Headers["X-Timestamp"];
    if (DateTime.UtcNow - DateTime.Parse(timestamp) > TimeSpan.FromMinutes(5))
        return BadRequest("Timestamp too old");
    
    // Check event ID (prevent duplicate processing)
    var eventId = Request.Headers["X-Event-ID"];
    if (await _db.IsProcessedAsync(eventId))
        return Ok();  // Already processed, but return 200
    
    // Queue for async processing
    await _queue.EnqueueAsync(new ProcessWebhookJob { Payload = body });
    
    return Accepted();  // 202 Accepted
}
```

## 🧪 Integration Testing

**Purpose**: Test components working together (APIs, databases, services).

### Contract Testing

**Idea**: Provider (API) and consumer (client) agree on contract.

```
Provider: Implements API
  ├─ GET /orders returns: { orderId, amount, status }
  └─ Provider test: Verify response matches contract

Consumer: Uses API
  ├─ Calls GET /orders
  └─ Consumer test: Verify can parse response
  
Contract test: Both sides agree on shape
```

**Example**:
```csharp
// Provider test (API)
[Test]
public async Task CreateOrder_ReturnsValidContract()
{
    var response = await _api.PostAsync("/orders", new { amount = 100 });
    var order = await response.Content.ReadAsAsync<OrderResponse>();
    
    // Verify contract
    Assert.That(order.OrderId, Is.Not.EqualTo(Guid.Empty));
    Assert.That(order.Status, Is.EqualTo("Created"));
}

// Consumer test (client)
[Test]
public async Task CreateOrder_ParsesResponseCorrectly()
{
    var response = await _client.CreateOrderAsync(100);
    
    // Verify consumer can handle response
    Assert.That(response.OrderId, Is.TypeOf<Guid>());
    Assert.That(response.Status, Is.EqualTo("Created"));
}
```

### E2E Testing

**Idea**: Test critical workflows end-to-end (UI, API, database).

```csharp
[Test]
public async Task CompleteOrderWorkflow()
{
    // Step 1: Navigate to order form
    await _browser.GotoAsync("https://app/orders/new");
    
    // Step 2: Fill form and submit
    await _browser.FillAsync("input[name='amount']", "100");
    await _browser.ClickAsync("button[type='submit']");
    
    // Step 3: Wait for order created
    await _browser.WaitForSelectorAsync("text=Order Created", new { Timeout = 5000 });
    
    // Step 4: Pay order
    await _browser.ClickAsync("button:has-text('Pay Now')");
    
    // Step 5: Verify payment successful
    await _browser.WaitForSelectorAsync("text=Payment Successful", new { Timeout = 5000 });
}
```

### Test Data

```csharp
// Factory pattern (create test data)
public class TestOrderFactory
{
    public static Order Create(decimal amount = 100)
    {
        return new Order
        {
            Id = Guid.NewGuid(),
            CustomerId = Guid.NewGuid(),
            Amount = amount,
            CreatedAt = DateTime.UtcNow
        };
    }
}

// Seed & cleanup
[SetUp]
public async Task SetupAsync()
{
    _testOrder = TestOrderFactory.Create();
    await _db.Orders.AddAsync(_testOrder);
    await _db.SaveChangesAsync();
}

[TearDown]
public async Task CleanupAsync()
{
    _db.Orders.Remove(_testOrder);
    await _db.SaveChangesAsync();
}
```
